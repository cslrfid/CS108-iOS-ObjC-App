  • subscriptions

    subscriptions is a dictionary of NSNumber instances indicating the MQTTQoSLevel.

    • The keys are topic filters.
    • The SessionManager subscribes to the given subscriptions after successfull (re-)connect
    • according to the cleansession parameter and the state of the session as indicated by the broker.
    • Setting a new subscriptions dictionary initiates SUBSCRIBE or UNSUBSCRIBE messages by SessionManager
    • by comparing the old and new subscriptions.

  • effectiveSubscriptions

  • -initWithPersistence:maxWindowSize:maxMessages:maxSize:maxConnectionRetryInterval:connectInForeground:streamSSLLevel:queue:

    initWithPersistence sets the MQTTPersistence properties other than default

    • - parameter: persistent YES or NO (default) to establish file or in memory persistence.
    • - parameter: maxWindowSize (a positive number, default is 16) to control the number of messages sent before waiting for acknowledgement in Qos 1 or 2. Additional messages are stored and transmitted later.
    • - parameter: maxSize (a positive number of bytes, default is 64 MB) to limit the size of the persistence file. Messages published after the limit is reached are dropped.
    • - parameter: maxMessages (a positive number, default is 1024) to limit the number of messages stored. Additional messages published are dropped.
    • - parameter: maxRetryInterval The duration at which the connection-retry timer should be capped. When MQTTSessionManager receives a ClosedByBroker or an Error event, it will attempt to reconnect to the broker. The time in between connection attempts is doubled each time, until it remains at maxRetryInterval. Defaults to 64 seconds.
    • - parameter: connectInForeground Whether or not to connect the MQTTSession when the app enters the foreground, and disconnect when it becomes inactive. When NO, the caller is responsible for calling -connectTo: and -disconnect. Defaults to YES.
    • - parameter: streamSSLLevel an NSString containing the security level for read and write streams
    • For list of possible values see:
    • https://developer.apple.com/documentation/corefoundation/cfstream/cfstream_socket_security_level_constants
    • Please also note that kCFStreamSocketSecurityLevelTLSv1_2 is not in a list
    • and cannot be used as constant, but you can use it as a string value
    • defaults to kCFStreamSocketSecurityLevelNegotiatedSSL
    • - parameter: queue Queue for MQTTSession.
    • - returns: the initialized MQTTSessionManager object

  • -connectTo:port:tls:keepalive:clean:auth:user:pass:will:willTopic:willMsg:willQos:willRetainFlag:withClientId:securityPolicy:certificates:protocolLevel:connectHandler:

    Connects to the MQTT broker and stores the parameters for subsequent reconnects

    • - parameter: host specifies the hostname or ip address to connect to. Defaults to @localhost.
    • - parameter: port specifies the port to connect to
    • - parameter: tls specifies whether to use SSL or not
    • - parameter: keepalive The Keep Alive is a time interval measured in seconds. The MQTTClient ensures that the interval between Control Packets being sent does not exceed the Keep Alive value. In the absence of sending any other Control Packets, the Client sends a PINGREQ Packet.
    • - parameter: clean specifies if the server should discard previous session information.
    • - parameter: auth specifies the user and pass parameters should be used for authenthication
    • - parameter: user an NSString object containing the user’s name (or ID) for authentication. May be nil.
    • - parameter: pass an NSString object containing the user’s password. If userName is nil, password must be nil as well.
    • - parameter: will indicates whether a will shall be sent
    • - parameter: willTopic the Will Topic is a string, may be nil
    • - parameter: willMsg the Will Message, might be zero length or nil
    • - parameter: willQos specifies the QoS level to be used when publishing the Will Message.
    • - parameter: willRetainFlag indicates if the server should publish the Will Messages with retainFlag.
    • - parameter: clientId The Client Identifier identifies the Client to the Server. If nil, a random clientId is generated.
    • - parameter: securityPolicy A custom SSL security policy or nil.
    • - parameter: certificates An NSArray of the pinned certificates to use or nil.
    • - parameter: protocolLevel Protocol version of the connection.
    • - parameter: connectHandler Called when first connected or if error occurred. It is not called on subsequent internal reconnects.

    Parameters

    the data to be sent. length may range from 0 to 268,435,455 - 4 - lengthof-topic bytes. Defaults to length 0.
    topic

    the Topic to identify the data
    retainFlag

    if YES, data is stored on the MQTT broker until overwritten by the next publish with retainFlag = YES
    qos

    specifies the Quality of Service for the publish qos can be 0, 1, or 2.

    Return Value

    the Message Identifier of the PUBLISH message. Zero if qos 0. If qos 1 or 2, zero if message was dropped
